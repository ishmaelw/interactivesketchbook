Micro Project Ideas

Premium Required

Issue / behavior / system:
Digital paywalls and systems that gate basic functionality behind payment.

Interaction rule:
The user can begin interacting freely at first, but once they reach a threshold of use, access becomes restricted. Actions that previously worked now trigger a “premium required” state. The restriction appears arbitrary rather than clearly rule-based.

Input:
Repeated clicks and scrolling attempts.

What changes over time:
Allowed actions decrease; locked elements increase; access shrinks.

Where friction appears:
Limited scrolling, blocked actions, and partial visibility of content.

Intended feeling:
Exclusion and frustration.

Verify You’re Human

Issue / behavior / system:
Automated systems that force people to repeatedly prove authentication to access digital space.

Interaction rule:
The user must complete a verification task before proceeding. Each time they succeed, a new verification appears. The criteria subtly change or become more ambiguous over time, even though the task appears similar.

Input:
Clicking, selecting, or completing small challenges.

What changes over time:
The number of verifications increases; instructions become less clear; tolerance for correctness shrinks.

Where friction appears:
Repetition, unclear rules, near-success followed by failure, and invisible evaluation.

Intended feeling:
Suspicion and humiliation.

Are You Still There?

Issue / behavior / system:
Attention-tracking systems that monitor inactivity and assume disengagement.

Interaction rule:
The system constantly monitors user activity. If there is even a brief pause, it triggers a warning asking if the user is still present. Over time, the allowed inactivity window shrinks, requiring constant interaction.

Input:
Mouse movement or key presses.

What changes over time:
Tolerance for stillness decreases; warnings appear faster; timeouts become more aggressive.

Where friction appears:
Punishment for stillness, forced micro-movements, constant monitoring.

Intended feeling:
Anxiety and surveillance.

Micro-Project — Progress #1

Verify You’re Human

1. Concept Summary

This interaction simulates a system that forces certain users to complete a human verification process before accessing content. The user clicks to attempt verification, but the outcome is not fully determined by their action. A hidden system variable assigns different levels of scrutiny. Over time, repeated attempts increase suspicion, making it harder to exit the verification state.

2. Primary Variable

let stage = 0;

This variable represents how many verification steps remain before access is granted. It changes based on system judgment, increasing with failure or inactivity and decreasing with successful attempts. It controls whether the user remains in verification or reaches the final access state.

3. Base Build

The first version includes one variable (stage), one input (mouse click), and two visible states.

State 1: Access Granted
The screen displays “Access Granted.” No verification is required.

State 2: Verification Required
The screen displays a verification prompt. Clicking attempts verification, but success is not guaranteed.

4. Structural Growth

Over time, the system evolves by adding escalation and instability. A stage variable tracks verification difficulty and increases scrutiny. Time-based behavior introduces suspicion for inactivity. Tolerance for correctness shrinks, and probabilistic failure introduces inconsistency. The system becomes less predictable and more restrictive.

Micro Project — Progress #2

The system introduces a randomized starting stage that determines whether the user must complete verification or is granted immediate access. A grid-based verification appears, requiring users to select correct squares. Successful attempts reduce the stage until access is granted.

The stage variable controls difficulty by changing grid size and verification steps. User interaction directly affects progression through the system.

Micro Project — Progress #3

Added:

  • Idle suspicion

  • Shrinking tolerance

Reflection:
The system shifted away from user skill and toward algorithmic judgment, making outcomes feel unpredictable and harder to trust.

Micro Project — Progress #4

Current Version:

The grid system has been removed, leaving only a pixelated background. Verification attempts are now opaque, with no visible criteria for success. The user can only click and wait for a response, reinforcing uncertainty and lack of control. The stage variable still controls difficulty and access, while the system maintains restriction and escalation.

Across versions, the system moved from a solvable task to an unclear and impossible process, shifting the focus from user ability to system control.
