Interface System Evaluation

Navigation Model & System Coherence (20 pts)

Score: 17/20

The navigation system is clear and mostly consistent across mobile and desktop. The mobile layout uses a bottom navigation bar (Home, MP, SB, Audit), aligning with common mobile interaction patterns. On desktop, this shifts to a left sidebar with expanded labels, maintaining the same conceptual structure.

However, there is one visible active states which is highlighting the button of which page you are on. The “Home” label at the top feels slightly disconnected from the main navigation system, I wish I could figure out a way to make the Microproject and Sketchbook icons like the home button.

Hierarchy & Mobile-First Logic (20 pts)

Score: 19/20

The mobile layout establishes a strong hierarchy with a clear header, dominant content area, and secondary navigation. The desktop version expands this structure intentionally by relocating navigation to the sidebar and scaling content appropriately.

This demonstrates true mobile-first design—building from constraint and scaling up. The only limitation is that the mobile layout is extremely minimal, which slightly reduces hierarchy depth.

Hierarchy Survival Across Contexts (15 pts)

Score: 12/15

The hierarchy remains consistent across both mobile and desktop. Content remains dominant, navigation stays secondary, and the header anchors the interface.

However, interaction cues are limited. There are no hover states, active states, or system feedback indicators. The use of placeholder “Content” blocks also makes it difficult to fully evaluate real-world hierarchy performance.

Theoretical Integration (15 pts)

Score: 11/15

The Interface System Statement references key ideas from Norman and Nielsen, including affordances, signifiers, consistency, and recognition over recall. These are partially visible in the design through labeled navigation and clear structure.

Key principles like system feedback, error prevention, and visibility of system status are not strongly demonstrated in the interface itself.

Cross-Page Consistency (10 pts)

Score: 8/10

The mobile and desktop layouts clearly belong to the same system, sharing consistent structure, labels, and layout logic.

I made sure to keep a home button displayed at all times throughout each page so the user can easily navigate back to the main screen.

Risk & Structural Ambition (5 pts)

Score: 3/5

The design uses standard UI conventions such as bottom navigation and sidebar layouts. While effective, it does not explore more experimental or complex interaction patterns.

Iterative Development (10 pts)

Score: 9/10

Through the few weeks of class work, the design has evolved significantly. Initial sketches have been refined into more cohesive layouts, and feedback has been incorporated to improve usability. Example: Enlarging the navigation section for mobile, making it easier to click and navigate.

Participation & Critique Engagement (5 pts)

Score: 4/5

The design demonstrates thoughtful decision-making and reflection. However, explicit evidence of critique integration or feedback-driven changes is not clearly shown.

Final Score: 90 / 100
