Movement between images is controlled by the left and right arrow keys using the keyPressed() function. Adding Auto Mode changed the behavior of the system by introducing timed progression, which shifts the interaction from manual control to automated movement. The currentIndex variable determines which image is displayed, and it updates based on keyboard input. The system uses wrap logic so that moving past the last image loops back to the first, and moving before the first loops to the last. While building this, I learned how important it is to separate state from display logic... the image doesn’t change randomly. it changes because the state variable changes. This made navigation feel intentional and controlled rather than reactive or accidental.
